Four pillars of Object oriented programming.

In this section , let's understand the four pillars of object oriented programming.

  • Abstraction
  • Inheritance
  • Polymorphism
  • Encapsulation

Abstraction

Abstraction , also called data hiding, stands for showing only essential details to the user , hiding unnecessary details of which user is not concerned about.Take the example of DVD player. You only have the buttons visible outside for various functions like power on , next previous etc. But the how these buttons work actually inside the box is hidden from you.

Inheritance

Inheritance as the name suggest, stands for deriving or inheriting properties of parent class into child classes. In C#, there are four types of inheritance

  • Single Inheritance- Only one child class inherits properties of parent class.
  • Multilevel Inheritance- In this inheritance, a subclass inherits parent class and similarly it becomes parent class for other child classes.
  • Multiple Inheritance- Multiple inheritance in c# is achieved via interfaces.
  • Hirarchical Inheritance- In this type of inheritance , multiple child classes inherit from only one parent class.

Polymorphism

Polymorphism stands for ability of an object to take many forms. Depending on the class , the object can behave in many ways. The best example is method overloading and method overriding.It means defining the method having same syntax in both parent and child class but behvior is accoding to the respective classes.

Encapsulation

Encapsulation is the concept of wrapping up data and object into single unit. The best example is class. We define all object, properties methods and fields into a class. It also provides security as we can declare our members of class as private, making it invisible from rest of the world.
